1. Checking for mandatory function parameters

JavaScript allows you to define default values for function parameters. Knowing this, we can implement an interesting way to check for mandatory parameters when calling functions and methods:

const isRequired = () => { throw new Error('param is required'); }; const print = (num = isRequired()) => { console.log(`printing ${num}`) }; print(2);//printing 2 print()// error print(null)//printing null
2. Formatting JSON code

You're probably very familiar with the JSON.stringify() method. And did you know that you can use this method to format JSON code? It's actually very simple.

The stringify() method takes three parameters. They are value, replace, and space. The last two parameters in this list are optional. That's why you may not have used them before. To configure JSON code alignment you should use the space parameter.

console.log(JSON.stringify({name:"John",Age:23},null,'\t')); >>> { "name": "John", "Age": 23 }
3. Extract unique values from an array

In the past, if you wanted to extract unique values from an array, you had to use the filter() method, which filters out duplicate values. Now you can use a new standard JavaScript object of type Set to do that. This is much easier and more convenient than filtering an array.

let uniqueArray = [...new Set([1, 2, 3, 3,3,"school","school",'ball',false,false,true,true])]; >>> [1, 2, 3, "school", "ball", false, true]
4. Removing false values from an array

It happens that you need to remove false values from an array. These are the values which in JavaScript are set to false. There are only 6 such values in the language (including false itself):

  • undefined
  • null
  • NaN
  • 0
  • "" (empty string)
  • false

Such values are easiest to filter out of an array using the following construction:

myArray .filter(Boolean);

If you want to modify the original array in some way, and then filter the new array afterwards, you can try something like the following code: When using such a construction, keep in mind that the original array, myArray, remains unchanged.

myArray .map(item => { // Выполнить изменения и вернуть новый элемент }) .filter(Boolean);

5. Combining several objects into one

I have had several cases where it was necessary to merge two or more objects. In such cases, I usually use the following technique:

const user = { name: 'John Ludwig', gender: 'Male' }; const college = { primary: 'Mani Primary School', secondary: 'Lass Secondary School' }; const skills = { programming: 'Extreme', swimming: 'Average', sleeping: 'Pro' }; const summary = {...user, ...college, ...skills};

The three dots used in the last line of this example are the spread operator.

6. Sorting numeric arrays

JavaScript arrays have a standard sort() method. It, by default, converts the array elements into strings and performs lexicographic sorting. This can cause problems when trying to sort numeric arrays. Such problems, however, are very easy to prevent:

[0,10,4,9,123,54,1].sort((a,b) => a-b); >>> [0, 1, 4, 9, 10, 54, 123]

A function that compares two elements of a numeric array is passed to the sort() method. Thanks to this function, the numeric array can be sorted as needed.

7. Disabling right-click action

You may want to prohibit users of your site from right-clicking context menus of items. Although it's not often needed, sometimes it can come in handy.

<body oncontextmenu="return false"> <div></div> </body>
8. Destructive assignment with variable renaming

The destructive assignment syntax is used in JavaScript expressions aimed at unpacking array elements or object properties into individual variables. Instead of using the existing names of object properties, the corresponding variables can be renamed:

const object = { number: 10 }; // Extract the value of the number property to a constant of the same name const { number } = object; // Extracting the value of the number property and assigning the variable name otherNumber const { number: otherNumber } = object; console.log(otherNumber); //10
9. Getting elements from the end of an array

If you need to get the elements at the end of the array, you can use the slice() array method, which is passed a negative number:

let array = [0, 1, 2, 3, 4, 5, 6, 7] console.log(array.slice(-1)); >>>[7] console.log(array.slice(-2)); >>>[6, 7] console.log(array.slice(-3)); >>>[5, 6, 7]
10. Waiting for the completion of all promises

It may happen that there is a need to wait for several promises to complete. In order to wait for several promises to complete, you can use the Promise.all() method. The main purpose of this method is to wait for the resolution of all the promises that have been passed to it.

const PromiseArray = [ Promise.resolve(100), Promise.reject(null), Promise.resolve("Data release"), Promise.reject(new Error('Something went wrong'))]; Promise.all(PromiseArray) .then(data => console.log('all resolved! here are the resolve values:', data)) .catch(err => console.log('got rejected! reason:', err))

When working with Promise.all() it is important to know that this method will throw an error if at least one of the promises it receives is rejected. This will cause Promise.all() not to wait for all the promises to finish.

But if you need to wait for all the promises to finish, regardless of whether they are successfully resolved or rejected, you should use the Promise.allSettled() method. This method is available in the final agreed version of the ES2020 standard.

const PromiseArray = [ Promise.resolve(100), Promise.reject(null), Promise.resolve("Data release"), Promise.reject(new Error('Something went wrong'))]; Promise.allSettled(PromiseArray).then(res =>{ console.log(res); }).catch(err => console.log(err)); //[ //{status: "fulfilled", value: 100}, //{status: "rejected", reason: null}, //{status: "fulfilled", value: "Data release"}, //{status: "rejected", reason: Error: Something went wrong ...} //]

Now, even though some promises are rejected, Promise.allSettled() returns the results of all promises.
